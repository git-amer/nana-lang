# Agent Protocol

- Read `git status --short` before editing so you understand what work already exists.
- Read `changelog.md` before making changes and preserve every documented change unless the user explicitly requests a rollback.
- Add or update the relevant dated entry in `changelog.md` whenever you make a code or documentation change.
- Do not overwrite, remove, or revert prior work just to make a new change fit. Adjust your edit around the existing work when possible.
- If a new request conflicts with documented work, stop and ask the user which direction to take before changing it.
- After finishing edits, update `changelog.md` with the final outcome and the verification you performed.
